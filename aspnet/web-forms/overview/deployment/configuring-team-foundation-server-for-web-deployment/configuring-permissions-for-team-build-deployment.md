## <a name="configuring-web-server-permissions"></a><span data-ttu-id="a0c64-119">Configurando permissões do servidor Web</span><span class="sxs-lookup"><span data-stu-id="a0c64-119">Configuring Web Server Permissions</span></span>

<span data-ttu-id="a0c64-120">Conforme descrito em [escolhendo a abordagem da direita para a implantação da Web](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md), há duas abordagens principais que você pode usar para implantar pacotes da web em um servidor web remoto:</span><span class="sxs-lookup"><span data-stu-id="a0c64-120">As described in [Choosing the Right Approach to Web Deployment](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md), there are two main approaches you can use if you want to deploy web packages to a remote web server:</span></span>

- <span data-ttu-id="a0c64-121">Implantar o aplicativo de um local remoto direcionando o *serviço de agente de implantação da Web* (também conhecido como o agente remoto) no servidor de destino.</span><span class="sxs-lookup"><span data-stu-id="a0c64-121">Deploy the application from a remote location by targeting the *Web Deployment Agent Service* (also known as the remote agent) on the destination server.</span></span>

<span data-ttu-id="a0c64-123">O agente remoto tem duas limitações importantes nesse caso:</span><span class="sxs-lookup"><span data-stu-id="a0c64-123">The remote agent has two key limitations in this case:</span></span>

<span data-ttu-id="a0c64-129">Por outro lado, a abordagem do manipulador de implantação da Web oferece várias vantagens:</span><span class="sxs-lookup"><span data-stu-id="a0c64-129">In contrast, the Web Deploy Handler approach offers various advantages:</span></span>

<span data-ttu-id="a0c64-137">No [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md) exemplo de solução, você especifica o tipo de autenticação (básica ou NTLM), as credenciais de implantação da Web e o endereço do ponto de extremidade (agente remoto ou manipulador de implantação da Web) no arquivo de projeto específicas do ambiente.</span><span class="sxs-lookup"><span data-stu-id="a0c64-137">In the [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md) sample solution, you specify the authentication type (basic or NTLM), the Web Deploy credentials, and the endpoint address (remote agent or Web Deploy Handler) in the environment-specific project file.</span></span> <span data-ttu-id="a0c64-138">Esses valores são usados para formular e executar um comando de implantação da Web quando o arquivo de projeto é executado.</span><span class="sxs-lookup"><span data-stu-id="a0c64-138">These values are used to formulate and run a Web Deploy command when the project file is executed.</span></span> <span data-ttu-id="a0c64-139">Para obter mais informações, consulte [Implantando pacotes da Web](../web-deployment-in-the-enterprise/deploying-web-packages.md).</span><span class="sxs-lookup"><span data-stu-id="a0c64-139">For more information, see [Deploying Web Packages](../web-deployment-in-the-enterprise/deploying-web-packages.md).</span></span>
