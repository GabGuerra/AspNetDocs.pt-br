## <a name="avoiding-cross-site-scripting-xss-attacks"></a><span data-ttu-id="78c17-152">Evitando ataques de Cross-Site XSS (script)</span><span class="sxs-lookup"><span data-stu-id="78c17-152">Avoiding Cross-Site Scripting (XSS) Attacks</span></span>

<span data-ttu-id="78c17-153">Sempre que você aceita a entrada do usuário e exibir novamente a entrada em seu site, você potencialmente abrir seu site a ataques de scripts entre sites (XSS).</span><span class="sxs-lookup"><span data-stu-id="78c17-153">Whenever you accept input from a user, and redisplay that input on your website, you potentially open your website to Cross-Site Scripting (XSS) Attacks.</span></span> <span data-ttu-id="78c17-154">Em teoria, um hacker mal-intencionado poderia enviar código JavaScript que é executado quando a entrada é exibida novamente.</span><span class="sxs-lookup"><span data-stu-id="78c17-154">In theory, a malicious hacker could submit JavaScript code that gets executed when the input is redisplayed.</span></span> <span data-ttu-id="78c17-155">O JavaScript pode ser usado para roubar senhas de usuário ou outras informações confidenciais.</span><span class="sxs-lookup"><span data-stu-id="78c17-155">The JavaScript could be used to steal user passwords or other sensitive information.</span></span>

<span data-ttu-id="78c17-156">Normalmente, você pode anular ataques XSS de codificação qualquer entrada que você se recuperar de um usuário antes de exibi-lo em uma página da web HTML.</span><span class="sxs-lookup"><span data-stu-id="78c17-156">Normally, you can defeat XSS attacks by HTML encoding whatever input you retrieve from a user before displaying it in a web page.</span></span> <span data-ttu-id="78c17-157">No entanto, a saída do Editor de HTML de codificação HTML não apenas codificaria &lt;script&gt; marcas, ele também deve codificar todas as marcas HTML.</span><span class="sxs-lookup"><span data-stu-id="78c17-157">However, HTML encoding the output of the HTML Editor would not only encode &lt;script&gt; tags, it would also encode all HTML tags.</span></span> <span data-ttu-id="78c17-158">Em outras palavras, você perderia toda a formatação, como o tipo de fonte, tamanho da fonte e cor do plano de fundo.</span><span class="sxs-lookup"><span data-stu-id="78c17-158">In other words, you would lose all of the formatting such as the font type, font size, and background color.</span></span>

<span data-ttu-id="78c17-159">Se você estiver coletando informações confidenciais de seus usuários, como senhas, números de cartão de crédito e números do seguro social - você não deve exibir o conteúdo não codificado recuperados de um usuário com o Editor de HTML.</span><span class="sxs-lookup"><span data-stu-id="78c17-159">If you are collecting sensitive information from your users -- such as passwords, credit-card numbers, and social security numbers - then you should not display un-encoded content that you retrieve from a user with the HTML Editor.</span></span> <span data-ttu-id="78c17-160">Você deve usar o Editor de HTML somente em situações em que você não exibir novamente o conteúdo HTML, ou o conteúdo HTML que está sendo enviado para seu site por uma parte confiável.</span><span class="sxs-lookup"><span data-stu-id="78c17-160">You should use the HTML Editor only in situations in which you are not redisplaying the HTML content, or the HTML content is being submitted to your website by a trusted party.</span></span>
